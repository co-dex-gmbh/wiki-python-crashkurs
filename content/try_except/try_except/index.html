    Exception Handling in Python¶

    Beim Programmieren in Python ist es wichtig, Ausnahmen ordnungsgemäß zu behandeln, um robusten und fehlerresistenten Code zu schreiben. Ausnahmen sind Ereignisse, die während der Ausführung eines Programms auftreten und den normalen Ablauf unterbrechen.

    Du kannst durch korrektes Handling von Ausnahmen Fehler auf elegante Weise bewältigen und so die Gesamtzuverlässigkeit deines Codes verbessern.

    Grundlagen des Exception Handlings¶

    Try-Except-Block¶

    Die Blöcke try und except werden verwendet, um Ausnahmen in Python zu behandeln. Der try-Block enthält den Code, der eine Ausnahme auslösen könnte, und der except-Block gibt an, wie die Ausnahme behandelt werden soll.

    1

    try:
    # Code, der eine Ausnahme auslösen könnte
    ergebnis = 10 / 0
except ZeroDivisionError as e:
    # Behandlung der spezifischen Ausnahme
    print(f"Fehler: {e}")

print("Ich lebe noch und kann weiter machen")

    Der gleiche Code ohne Exceptionhandling bricht die Durchführung des Programms ab:

    1
2
3
    ergebnis = 10 / 0

print("Das wirst du niemals sehen")

    class PhoneNumberNotFoundError(Exception):
    pass


phone_numbers = {'Jürgen': '01234-5678', 'Monika': '+49-156-89345'}


def call_by_phone_number(name):
    if name not in phone_numbers:
        raise PhoneNumberNotFoundError(f"There is no Phonenumber for {name}")

try:
    n = input('Wen willst du anrufen?')
    call_by_phone_number(n)
except PhoneNumberNotFoundError as e:

    Der else-Block wird ausgeführt, wenn keine Ausnahmen im try-Block ausgelöst werden.

except ZeroDivisionError as e:
    print(f"Fehler: {e}")
else:
    print(f"Ergebnis: {ergebnis}")

    ⚠ Achtung: Exception Handling und Ablaufsteuerung¶

    Es ist wichtig zu betonen, dass Exception Handling nicht als Mechanismus zur Ablaufsteuerung genutzt werden sollte. Ausnahmen sollten nicht für die Kontrolle des normalen Programmflusses verwendet werden, sondern ausschließlich für die Behandlung von unerwarteten Ereignissen und Fehlerzuständen.

    class NegativeNumberError(Exception):
    pass

def check_positive(numbers):
    for number in numbers:
        if number < 0:
            raise NegativeNumberError(f"Zahl {number} ist negativ")
        print(f"Zahl {number}")

# Beispielaufruf
try:
    check_positive([1, 2, 0, -2, 3])
except NegativeNumberError as e:
    print(e)

    def get_number(prompt="Gib eine Ganzzahl ein: "):
    while True:
        try:
            return int(input(prompt))
        except ValueError:
            print("Das war keine erlaubte Zahl. Erneut bitte:")

def get_operation(prompt="Gebe Operation ein: "):
    operations = {"+": add, "-": subtract, "*": multiply, "/": divide}
    while True:
        op = input(prompt)
        if op in operations:
            return operations[op]
        print(f"{op} ist keine der erlaubten Operationen {list(operations.keys())}. Erneut bitte:")

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

def divide(a, b):
    return a / b

while True:
    num1 = get_number("Gib erste Ganzzahl ein: ")
    num2 = get_number("Gib zweite Ganzzahl ein: ")
    operation = get_operation()

    try:
        result = operation(num1, num2)
    except Exception as e:
        print(f"Operation nicht möglich: {e}")
    else:
        print(f"Das Ergebnis ist: {result}")

    if input("Möchtest du eine weitere Berechnung durchführen? (j/n): ").lower() != 'j':
        break
    Übertriebene Rekursion

    Zu welchem Fehler führt der folgende Code und warum?

    1
2
3
4
    def fak(n):
    return 1 if n <= 1 else n * fak(n-1)

print(fak(1000))
    Lösung

    Der Code führt zu einem RecursionError.

    Erklärung: Die Funktion fak ruft sich immer wieder selbst auf (das nennt man eine rekursive Funktion). Der Callstack wird damit immer größer, bis dieser irgendwann die maximale Größe überschreitet. Dann wird ein RecursionError geworfen.

    Um dies zu demonstrieren und abzufangen, können wir den Code wie folgt anpassen:

    1
2
3
4
5
6
7
8
    def fak(n):
    return 1 if n <= 1 else n * fak(n-1)

try:
    print(fak(1000))
except RecursionError as e:
    print(f"RecursionError aufgetreten: {e}")
    print("Die Rekursionstiefe war zu groß für diese Berechnung.")

    Eine bessere Implementierung für große Zahlen wäre eine iterative Lösung oder die Verwendung des math.factorial() Moduls.
